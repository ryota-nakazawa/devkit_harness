# learnings/

**単発の学び(草稿)**。各案件のフェーズ8(検証)とフェーズ9(改善方針)で生まれた気付きを溜める場所。

## 役割

`recipes/` の手前にある **草稿エリア**。
ここに2回以上同じパターンが現れたら `recipes/` に昇格する。

## ファイル形式

```
learnings/
└── <YYYYMMDD-HHMMSS>-<project-slug>.md
```

## 中身

```markdown
---
project: <project-slug>
domain: <業界カテゴリ>
created_at: <ISO8601>
status: draft | promoted | rejected
---

## 何をやったか
1段落

## 刺さったこと
- 箇条書き(必須:1つ以上)

## 外したこと
- 箇条書き(必須:1つ以上 — "特になし" でも明記する)

## 次回試したいこと
- 任意

## 関連レシピ候補
- 既存の `recipes/<tag>/` に近ければそのタグを書く
- 新規パターンと思えば仮タグを書く
```

## 重要

- **失敗の記録を必須化**する。成功例ばかり溜めると生存者バイアスで判断を誤る
- ここに書かれただけでは "確定知" ではない。`recipes/` への昇格を経て初めて適用候補になる
