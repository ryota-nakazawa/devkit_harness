# Phase 4: 入出力整理

## 入力
- `projects/<slug>/phase-1-requirements.md`
- `projects/<slug>/phase-3-scenarios.md`

## 処理手順

### 1. 入力項目
ユーザーが何を入力するかを項目単位で整理。型・必須/任意・例も明記。

### 2. 出力項目
画面に何が表示されるか、ファイル/メールで何が出るか。表示単位(1件/一覧/集計)も明記。

### 3. データの流れ
入力 → 処理 → 出力 の流れを矢印図か箇条書きで。
データがどこに保存されるか(ローカル/メモリ/外部)も明記。

### 4. ダミーデータ設計
プロトに使うダミーデータの仕様を決める。リアリティを上げるための注意:
- 件数は十分に(空っぽに見えない)
- 名前/日付は **生々しいもの** を使う(「顧客A」より「山田太郎」)
- エッジケース(空欄/長い文字列)を1〜2件混ぜる

### 5. 外部API判定
`external_api` が必要かここで判定。必要なら `auth: none|key` と `mock` を決める。
キー必須なら mock のみ使用(キー漏洩防止)。

## 出力

`projects/<slug>/phase-4-io.md`:

```markdown

---

# Phase 4: 入出力定義

## 出力項目

2. <処理> →

保存先: ローカルストレージ / メモリ / 外部API

## ダミーデータ仕様
- 件数: <N>件
- 命名規則: <例>
- エッジケース: <内容>
```
